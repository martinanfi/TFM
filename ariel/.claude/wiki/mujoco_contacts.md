# mujoco_contacts

Contact detection and force extraction in MuJoCo. Contacts are generated by the position stage of forward/inverse dynamics and stored in `data.contact`.

## Contact Storage

`data.contact` is an array of `mjContact` structs, sorted such that:
- Multiple contacts between the same pair of bodies are contiguous.
- Body pairs are sorted (first body = major index, second body = minor index).

Not all detected contacts enter the force computation. Active contacts have `contact.exclude == 0` and a valid `contact.efc_address`.

**Reasons for exclusion:**
- The geom's `gap` attribute
- Internal processing (virtual contacts for intermediate computations)

## Key mjContact Fields

| Field | Type | Description |
|-------|------|-------------|
| `dist` | `float` | Contact distance (negative = penetration) |
| `pos` | `(3,)` | Contact position in world frame |
| `frame` | `(9,)` | Contact frame axes — rows (not columns!): [0–2] = normal (X), [3–5] = Y, [6–8] = Z |
| `geom1`, `geom2` | `int` | Geom IDs of the two contacting geoms |
| `efc_address` | `int` | Index into `efc_*` arrays; -1 if excluded |
| `exclude` | `int` | 0 = active, nonzero = excluded |
| `dim` | `int` | Contact dimensionality (condim) |

> **Important:** `mjContact.frame` stores axes as **rows** (row-major), unlike all other frame matrices in MuJoCo which store axes as columns. The contact normal is frame[0:3].

## Extracting Contact Forces

```python
import mujoco
import numpy as np

model = mujoco.MjModel.from_xml_path('model.xml')
data  = mujoco.MjData(model)

mujoco.mj_step(model, data)

# Iterate over active contacts
for i, contact in enumerate(data.contact):
    if contact.efc_address < 0:
        continue  # excluded contact

    # Raw constraint force (interpretation depends on friction cone type)
    raw_force = data.efc_force[contact.efc_address : contact.efc_address + contact.dim]

    # Convert to intuitive 3D force + 3D torque in contact frame
    result = np.zeros(6)
    mujoco.mj_contactForce(model, data, i, result)
    contact_force  = result[:3]   # force in contact frame
    contact_torque = result[3:]   # torque (nonzero only when condim > 3)
```

## mj_contactForce

```python
mujoco.mj_contactForce(model, data, contact_id, result)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `model` | `MjModel` | |
| `data` | `MjData` | Must have been stepped (efc_force populated) |
| `contact_id` | `int` | Index into `data.contact` |
| `result` | `ndarray (6,)` | Output: [fx, fy, fz, tx, ty, tz] in contact frame |

Force and torque are expressed in `mjContact.frame`. Torque is zero when `condim` ∈ {1, 3}.

## Friction Cone Types

Use `mujoco.mj_isPyramidal(model)` to check which friction cone is active:
- **Pyramidal**: force components along redundant non-orthogonal pyramid-edge axes. Use `mj_contactForce` to interpret.
- **Elliptic**: force in standard contact-frame coordinates.

## Custom Collision Functions

To override built-in collision functions (e.g. for non-convex meshes):

```python
# mjCOLLISIONFUNC is a 2D array indexed by geom type pair
# Set to a custom function with signature: mjfCollision(model, data, g1, g2, margin, con)
# After detecting contacts, call mj_addContact to add them to data.contact
mujoco.mjCOLLISIONFUNC[type1][type2] = my_collision_fn
```

Custom functions must construct `mjContact` structs and call `mj_addContact`. They are called after broad-phase (near-phase only).

## Notes

- `data.contact` is regenerated every step; do not hold references across steps.
- Contact list length is `data.ncon` (number of detected contacts, including excluded ones).
- For contact counting and debugging: `sum(1 for c in data.contact if c.efc_address >= 0)` gives active count.
- Contact normal force magnitude: `result[0]` from `mj_contactForce` (X-axis of contact frame = normal).
